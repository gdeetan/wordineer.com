Overview

The Wordineer API is a free, JSON-based REST API that gives developers programmatic access to word generation, text analysis, and language utility tools. It is designed to be simple enough to call from a cURL command and powerful enough to build production applications.

Base URL

https://api.wordineer.com/v1

Response Format

All responses are JSON. Successful responses follow this envelope:

{
  "ok": true,
  "data": { ... },
  "meta": {
    "endpoint": "/random-word",
    "version": "1.0"
  }
}

Authentication

Basic tier (up to 100 requests/day): No API key required. Simply call the endpoint directly.

Standard tier (up to 5,000 requests/day): Free registration required. Pass your API key in the request header: 

X-API-Key: wdn_your_api_key_here

To get a free API key, email api@wordineer.com with the subject line "API Key Request". We'll respond within 1–2 business days.

Note: API keys are for server-side use only. Do not expose your key in client-side JavaScript or public repositories.

Rate Limits

Tier Daily Limit Burst (per minute) Key Required
Basic (anonymous) 100 requests 10 req/min No
Standard (free key) 5,000 requests 60 req/min Yes

Exceeding the rate limit returns a 429 Too Many Requests response. The Retry-After header will indicate when you may resume requests.

Endpoints

GET /random-word

Returns one or more random English words, with optional filters.

Query Parameters

ParameterTypeDefaultDescription
count integer 1 Number of words to return (max 100).
type string any Filter by part of speech: noun, verb, adjective, adverb.
min_length integer 1 Minimum word character length.
max_length integer 20 Maximum word character length.
starts_with string none Return only words starting with this letter or string.

Example Request

curl "https://api.wordineer.com/v1/random-word?count=3&type=noun&min_length=5"

Example Response

{
  "ok": true,
  "data": {
    "words": ["harbor", "lantern", "century"]
  },
  "meta": {
    "count": 3,
    "filters": { "type": "noun", "min_length": 5 }
  }
}

POST /word-count

Analyzes a block of text and returns word, character, sentence, and paragraph counts.

Request Body

{
  "text": "Your text goes here. It can be multiple sentences."
}

Example Request

curl -X POST "https://api.wordineer.com/v1/word-count" \
  -H "Content-Type: application/json" \
  -d '{"text": "The quick brown fox jumps over the lazy dog."}'

Example Response

{
  "ok": true,
  "data": {
    "words": 9,
    "characters": 44,
    "characters_no_spaces": 35,
    "sentences": 1,
    "paragraphs": 1,
    "reading_time_seconds": 4,
    "unique_words": 9
  }
}

GET /rhymes

Returns a list of words that rhyme with the given input word.

Query Parameters

ParameterTypeRequiredDescription
word string Yes The word to find rhymes for.
limit integer No Max number of rhymes to return (default: 10, max: 50).

Example Request

curl "https://api.wordineer.com/v1/rhymes?word=light&limit=5"

Example Response

{
  "ok": true,
  "data": {
    "word": "light",
    "rhymes": ["night", "sight", "right", "tight", "flight"]
  },
  "meta": { "count": 5 }
}

Error Codes

When a request fails, the API returns a non-2xx HTTP status code and an error envelope:

{
  "ok": false,
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "The 'count' parameter must be an integer between 1 and 100.",
    "param": "count"
  }
}
HTTP StatusError CodeMeaning
400INVALID_PARAMETERA query parameter or body field has an invalid value.
400MISSING_PARAMETERA required parameter was not provided.
401INVALID_API_KEYThe provided API key is invalid or revoked.
404WORD_NOT_FOUNDNo results match the given filters or input word.
429RATE_LIMIT_EXCEEDEDYou have exceeded the allowed request rate.
500INTERNAL_ERRORAn unexpected server error occurred. Please try again.
